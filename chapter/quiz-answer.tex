\subsection*{第2章 データベースの基礎知識}
\begin{itemize}
    \item \textbf{Q1}: 代表的なDBMSとしてMySQL，PostgreSQL，Oracle Databaseなどが挙げられ，これらはRDBMS（関係データベース）に分類される．それ以外（NoSQL等）のDBMSも存在する
    \item \textbf{Q2}: 線形探索では平均して $\frac{n+1}{2}$ 回の確認が必要となるため，100万件の場合は平均50万回となる．
    \item \textbf{Q3}: オンラインショッピング（購買・在庫管理），SNS（ユーザ管理），ブログ（記事管理）などが考えられる．
    \item \textbf{Q4}: CSVはカンマ区切り，TSVはタブ区切りのテキストファイル．表データを扱うために利用される．欠点として，大規模データ処理の困難さ，データ型や制約の欠如，同時アクセス制御の不備が挙げられる．
    \item \textbf{Q5}: データベース分野のチューリング賞受賞者として，Jim GrayやMichael Stonebrakerなどが該当する．
\end{itemize}

\subsection*{第5章 データベースの異状}
\begin{itemize}
    \item \textbf{Q1}: (1) 挿入時異状，(2) 更新時異状，(3) 削除時異状．
    \item \textbf{Q2}: 更新時異状の例: 特定の営業担当者の成績を変更する際，同じ担当者が複数行に存在する場合に一部のレコードの修正漏れが生じ，データに矛盾が発生する．
    \item \textbf{Q3}: 削除時異状の例: あるユーザの営業記録を削除すると，その商品や営業担当の連絡先情報まで意図せずデータベースから消滅してしまう．
    \item \textbf{Q4}: 挿入時異状の例: まだ誰も購入（ユーザとの関連）していない新しい商品やメーカーの情報を，テーブルに登録することができない．
\end{itemize}

    \item \textbf{Q6}: 図\ref{fig:erd-to-schema}のER図を変換ルールに従い，以下の6つの関係スキーマに変換する．

\end{itemize}

\subsection*{第7章 関係データベースの設計 - 正規化}
\begin{itemize}
    \item \textbf{Q1-1}: 同じ楽曲を評価するタプル間では「アーティスト」「収録アルバム」「ジャンル」が常に一致し，「スコア」はユーザごとに異なりうる例を5〜6件作成する．
    \item \textbf{Q1-2}: 省略
    \item \textbf{Q1-3}: 候補キー \{ユーザ名, 楽曲\} のもとで，以下の2ステップでBCNFへ分解する．
    \begin{enumerate}
        \item $FD_4: 収録アルバム \to アーティスト$で分解．\textbf{アルバム}(\underline{収録アルバム}, アーティスト)（BCNF），\textbf{残余スキーマ}（ユーザ名, 楽曲, 収録アルバム, ジャンル, スコア）（第1正規形）が得られる．
        \item $FD_{2,3}: 楽曲 \to 収録アルバム, ジャンル$で分解．\textbf{楽曲情報}(\underline{楽曲}, 収録アルバム, ジャンル)（BCNF），\textbf{評価}(\underline{ユーザ名, 楽曲}, スコア)（BCNF）が得られる．
    \end{enumerate}
    \item \textbf{Q2}: $FD_2$，$FD_3$，$FD_4$の順で分解しても，本文の例（$R_6 \sim R_9$）と同一のBCNFスキーマ集合が得られる（分解の順序は最終結果に影響しない）．各ステップで切り出したスキーマはBCNFとなり，残余スキーマは次のステップへ引き継がれる．
    \item \textbf{Q3}: 候補キーは $\{A, B\}$ と $\{A, C\}$ の2つ（$FD_1$: $A, B \to C$ と $FD_2$: $C \to B$ により，$A$ と $B$ または $C$ があれば全属性が決まるため）．$FD_5: E \to F$で分解後，$FD_{3, 4}: C \to D, E$で分解すると，$R_1(\underline{E}, F)$（BCNF），$R_2(\underline{C}, D, E)$（BCNF），$R_3(A, B, C)$（第3正規形）が得られる．$R_3(A, B, C)$ を $FD_2$ でさらに分解するとBCNFが得られるが，$FD_1: A, B \to C$が保持できなくなるため第3正規形で留める．
    \item \textbf{Q4}: $FD_1$，$FD_2$，$FD_3$の順で分解すると，すべてBCNFの以下の4スキーマが得られる：$R_1(\underline{\text{選手}}, \text{出身校})$，$R_2(\underline{\text{チーム}}, \text{チーム創立年})$，$R_3(\underline{\text{チーム, 年度}}, \text{監督})$，$R_4(\underline{\text{選手, 年度}}, \text{チーム})$．
\end{itemize}
